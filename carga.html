Implementación en C – Acceso O(1), encapsulamiento, carga desde archivo .txt

1. Estructura final del autómata

typedef struct {
    Tdata Q;                // conjunto de estados (cada uno Tdata STR)
    Tdata Sigma;            // alfabeto (conjunto de símbolos)
    Tdata** delta;          // matriz dinámica [num_estados][num_simbolos]
    int num_estados;
    int num_simbolos;
    int q0;                 // índice del estado inicial
    Tdata F;                // conjunto de estados finales
} Automata;
typedef Automata* AF;

delta es una matriz de Tdata. Cada celda delta[i][j] es un conjunto que guarda los nombres de los estados destino. Esto permite representar tanto AFD (un solo destino) como AFND (múltiples destinos).

2. Cambios estructurales clave respecto a la versión inicial

Versión inicial (lista de transiciones): se usaba un arreglo lineal de TransitionEntry con campos from, symbol, to. La consulta δ(qᵢ, aⱼ) requería una búsqueda secuencial O(n).

Versión final (matriz dispersa): se eliminó TransitionEntry y se incorporó Tdata** delta con dimensiones num_estados × num_simbolos. El acceso es directo por índices → O(1).

¿Cómo se garantiza O(1)? Una vez que convertimos el nombre de un estado a su índice (0,1,2…) y el símbolo a su índice, la operación aut->delta[estado][simbolo] se traduce en una indexación de arreglo de dos dimensiones, de costo constante.

3. Manejo de cadenas: TAD_str (lista enlazada de caracteres)

Todas las operaciones sobre nombres de estados/símbolos se realizan mediante funciones propias, no con la biblioteca estándar de cadenas C (strcmp, strcpy, etc.).

5. Carga desde archivo de texto (formato 5 líneas)

q0,q1,q2,q3
a,b
q0,a,q0;q0,a,q1;q0,a,q2;q0,b,q0;q1,a,q2;q2,b,q3;q3,a,q3;q3,b,q3
q0
q3
  • Línea 1: estados (separados por coma).
  • Línea 2: alfabeto (símbolos separados por coma).
  • Línea 3: transiciones. Cada transacción: origen,símbolo,destino. Las transacciones se separan por coma, y los destinos múltiples se separan con ; (ej: q0,a,q1;q2).
  • Línea 4: estado inicial.
  • Línea 5: estados finales (separados por coma).

🔹 Flujo de cargar_automata_desde_archivo

  1. fopen(ruta, "r") – abre el archivo.
  2. leer_conjunto_desde_archivo para poblar aut->Q (estados).
  3. leer_conjunto_desde_archivo para poblar aut->Sigma (alfabeto).
  4. inicializar_matriz(aut) – cuenta estados y símbolos, reserva delta y llena cada celda con create_set().
  5. procesar_linea_transiciones(aut, linea) – parsea la línea de transiciones.
  6. Lee estado inicial, limpia espacios, obtiene su índice con estado_a_indice y asigna a aut->q0.
  7. leer_conjunto_desde_archivo para poblar aut->F.
  8. fclose – cierra el archivo.

6. Conversión nombre ↔ índice

  • Nombre → índice: estado_a_indice(AF aut, str nombre) recorre el conjunto aut->Q usando obtener_data/obtener_next y compara con compare_str. Ídem para simbolo_a_indice sobre aut->Sigma.
  • Índice → nombre: str indice_a_str(Tdata set, int indice) recorre la lista hasta la posición indice y devuelve la cadena. Se usa en mostrar_automata para imprimir la tabla.

7. Agregar transición: agregar_transicion

void agregar_transicion(AF aut, str from, str symbol, str to) {
    int i = estado_a_indice(aut, from);
    int j = simbolo_a_indice(aut, symbol);
    int k = estado_a_indice(aut, to);
    if (i == -1 || j == -1 || k == -1) return;
    if (aut->delta == NULL) inicializar_matriz(aut);
    Tdata dest_node = create_str();
    dest_node->string = copy_str(to);
    insert_set(&(aut->delta[i][j]), dest_node);
}

Explicación paso a paso:

  1. Convierte los nombres a índices enteros (búsqueda en los conjuntos).
  2. Inicializa la matriz si aún no existe.
  3. Crea un nodo Tdata de tipo STR con una copia independiente de la cadena destino (copy_str).
  4. Inserta el nodo en el conjunto aut->delta[i][j] (si ya existe, insert_set evita duplicados).

10. Conclusión

El proyecto implementa completamente un autómata AFND/AFD con matriz dispersa de transiciones, logrando acceso O(1) y una representación clara. Se han diferenciado claramente las funciones de la biblioteca estándar (para archivos y parseo inicial) de las funciones propias del TAD_str (manejo de cadenas como listas enlazadas). La estructura final (matriz bidimensional) mejora drásticamente la eficiencia respecto a la versión con lista de transiciones. El código es modular, bien encapsulado y listo para ser extendido (conversión a AFD, minimización, etc.).
